# Architectural Design Requirements (ADR)

This directory contains Architectural Design Requirements (ADRs) for the CDP Portal service. ADRs are documents that
capture important architectural decisions made during the development of the software. They provide context, reasoning
and implications of these decisions to help current and future team members understand the architecture of the system.

## Requirements/Documentation

- [platform-state.md](platform-state.md): Describes the architecture and design decisions related to the platform state
  management within the CDP Portal service.
